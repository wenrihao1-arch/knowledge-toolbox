# 深研体系 v6.5（Domain Deep Research System）

> **定位：** 三兄弟之一——深读管文章精读，深析管事件分析，深研管领域研究。
> **融合的权威方法论：** 第一性原理、系统文献综述法 SLR、行业研究七大维度、波特五力、麦肯锡问题解决法、刻意练习、费曼技巧、多元思维模型。

---

## 一、适用范围

| 用户说什么 | 处理 |
|-----------|------|
| "研究一下XX" / "帮我了解XX领域" | 正式深研 |
| "深研一下XX" | 显式指令直通 |
| 直接问一个具体问题 | 通常走搜索收集或直接回答 |

---

## 研究价值评估（前置门控）

在投入四步法之前，先花30秒判断：这个领域值得系统研究吗？

| 评估项 | 问自己 |
|--------|--------|
| 信息可获得性 | 能找到权威信源吗？还是只能靠碎片信息？ |
| 认知杠杆 | 系统了解后能带来什么？ |
| 范围可控性 | 这个领域边界清晰吗？ |
| 时间性价比 | 投入的时间 vs 预期的认知收益，值吗？ |

**判定：**
- 高杠杆 + 边界清晰 + 信息充分 → 正式深研
- 信息不足 / 范围过大 → 降级处理

**降级路径：**
- 范围过大 → 建议聚焦子方向，或先用搜索收集快速了解核心脉络
- 信息不足 → 先用搜索收集整理目前能找到的信息
- 一次性好奇心 → 直接简要回答

## 二、深研四步法

### 第一步：领域地图

不急着深入，先看清整片森林。

**输出：**
- 知识分支地图（MECE：互不重叠、完全穷尽）
- 分支间的依赖关系
- 每个分支的投入深度建议
- 一句话回答：这个领域的"第一性原理"是什么？

---

### 第二步：信源+分析（迭代）

信源收集和分析解读同步进行，找到一篇读一篇，评估质量，决定是否继续。

**迭代循环：**
```
搜信源 → 读+分析 → 评估：够了吗？
                        ↓
                   是→进入第三步
                   否→回到"搜信源"继续迭代
```

**迭代上限：** 每 3 轮迭代后暂停，向用户汇报当前进度，让用户确认是否继续或调整方向。

**信源优先级：** 系统综述 > RCT/权威指南 > 学术论文 > 教科书 > 行业白皮书 > 自媒体

**信源筛选日志：**

每轮迭代结束时更新：

| 轮次 | 搜索词/来源 | 找到数 | 纳入数 | 排除原因 |
|------|------------|--------|--------|---------|
| 1 | [搜索词] | N | n | 过时/低证据层级/不相关 |
| 2 | | | | |

**分析工具（灵活选用）：**

| 工具 | 用于 | 适用 |
|------|------|------|
| CERIC | 拆解论证结构 | 通用 |
| 第一性原理 | 识别假设→拆到基本事实→重建 | 通用 |
| 多元思维模型 | 跨学科交叉验证 | 通用 |
| 逻辑谬误清单 | 识别推理缺陷 | 通用 |
| 肖璟七大维度 | 行业分析 | 行业型优先 |
| 波特五力 | 竞争格局 | 行业型优先 |
| PEST | 宏观环境 | 行业型优先 |

**多信源综合：Synthesis Matrix**

收集到3个以上信源后，做交叉对比：

| 子问题 | 信源A | 信源B | 信源C | 综合判断 |
|--------|-------|-------|-------|---------|
| 问题1 | 发现 | 发现 | 发现 | A和B一致，C不同，因为…… |
| 问题2 | 发现 | 发现 | 未讨论 | 二者都支持，但B证据更可靠 |

**原则：** 每个子问题至少让两个信源"对话"。

**边界处理：**
- 信源全部分歧 → 启动「争议升级路径」
- 某子问题仅1个信源覆盖 → 标记置信度为「低」
- 某子问题无信源覆盖 → 在「后续待挖」中列出

**争议升级路径：**
```
步骤①：检查证据层级——哪个信源的方法论和证据质量更高？
步骤②：寻找第三方仲裁——学术综述、行业协会共识、监管机构报告优先
步骤③：如果①②都无法解决 → 标记为「领域暂无共识」，列出各观点
```

**关键原则：** "暂无共识"本身就是一个有效的研究结论。比强行选一个更有价值。

---

### 第三步：综合输出

**输出内容：**
1. 深研成果卡片（见下方模板）
2. 完整的领域分析（上文下卡片）
3. 附：深读精读卡片（如有）

**完成度检查清单：**
- [ ] 领域地图：画出主要分支和依赖关系
- [ ] 信源筛选日志：关键搜索词和决策痕迹有记录
- [ ] 信源：至少找到3个权威信源
- [ ] Synthesis Matrix：至少填充了核心子问题
- [ ] 争议处理：遇到分歧时走了争议升级路径吗？
- [ ] 核心概念/原理拆解完毕
- [ ] 能一句话说清该领域的核心结论
- [ ] So what? 检查：每个关键发现都问了"这意味着什么？"

---

### 第四步：归档

归档到 `D:\知识库\资料库\`。

**升级来源记录：** 如果这次深研是被深析或搜索收集建议升级而来的，归档时注明启动来源。

**更新规则：** 不覆盖旧文件。有新结论时追加更新章节。

## 四、工具间协作

```
深研（研究领域）
  → 遇到高质量文章 → 调用深读（精读后附在附录）
  → 遇到相关新闻事件 → 调用深析
  → 需要搜索补充信息 → 调用搜索收集

深读（分析文章）
  → 涉及需要系统了解的领域 → 调用深研

深析（分析事件）
  → 发现值得系统了解的领域 → 建议升级到深研
```

**注意：** "调用"不是"切换"——完成子任务后回到原任务。

## 五、元认知检查

| 检查项 | 自问 |
|--------|------|
| 立场觉察 | 我内心希望这个结论是什么？ |
| 确认偏误 | 我是不是只找支持自己观点的信息？ |
| 权威迷信 | 信结论是因为它成立，还是因为它出自权威？ |
| 证据层级 | 结论的证据强度够吗？ |
