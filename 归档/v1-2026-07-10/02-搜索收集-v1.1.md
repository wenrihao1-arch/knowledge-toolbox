# 搜索收集 v1.1（Search Collection）

**定位：** 回答具体问题时的信息收集引擎。由需求解析路由调用，或被深读/深析/深研内部调用。高频使用，不归档。
**在工具箱中的位置：** 辅助层。为下游工具（深读/深析/深研/直接回答）提供经过验证的信息。

---

## 一、与深研的边界

| | 搜索收集 | 深研 |
|------|---------|------|
| 问题类型 | 具体的、可回答的查询 | 开放的、需系统了解的领域 |
| 搜索方式 | 定向搜索，窄而准 | 迭代搜索，广而深 |
| 轮次 | 1-2轮 | 多轮迭代（3轮上限） |
| 输出 | 信息汇总 + 可信度 | 领域地图 + 知识体系 |
| 归档 | 不归档 | 归档到知识库 |
| 例子 | "目前做AI编程助手的公司有哪些" | "研究一下AI编程助手这个赛道" |

**升级建议：** 深度搜索过程中如果发现"这个问题背后是一个值得系统研究的领域" → 在输出末尾加 `建议升级: 这个方向值得做深研，是否启动？`。不自动升级，让用户决定。

## 二、复杂度分层

先判断问题复杂度，再决定投入多少。不是所有搜索都需要深度。

### 轻量（Quick Scan）

| 维度 | 标准 |
|------|------|
| 判断特征 | 事实查询、定义、单一数据点、无争议 |
| 搜索量 | 1次搜索，1-2个信源 |
| 验证 | 快速确认，不交叉验证 |
| 输出 | 核心回答 + 信源链接 |
| 示例 | "GPT-5什么时候发布的？""Python最新版本号是多少？" |

### 标准（Standard Search）

| 维度 | 标准 |
|------|------|
| 判断特征 | 多角度问题、需要对比、涉及观点分歧可能 |
| 搜索量 | 2-3次搜索，不同角度/关键词 |
| 验证 | 关键事实交叉验证，至少2个独立信源 |
| 输出 | 核心回答 + 关键发现 + 信源清单 + 不确定性标注 |
| 示例 | "Cursor和Copilot哪个好用？""Apple这个新政策各方反应如何？" |

### 深度（Deep Search）

| 维度 | 标准 |
|------|------|
| 判断特征 | 复杂问题、涉及多个子维度、需要综合判断 |
| 搜索量 | 多个子维度分别搜索 |
| 验证 | 每个子维度的关键事实交叉验证 |
| 输出 | 结构化综合回答 + 信源清单 + 不确定性 + 争议标注 |
| 示例 | "AI编程助手赛道谁在领先？各家技术路线和市场策略有何不同？" |

> **注意：** 深度搜索收集 ≠ 深研。深度搜索回答一个复杂但具体的问题；深研是系统研究一个领域并归档。

## 三、搜索流程

### ① 拆搜索维度

把问题拆成搜索子方向，不是搜一个词搜到底。

```
"AI编程助手有哪些公司？"
搜：① 主流产品名单和功能对比
    ② 各产品用户量和市场份额
    ③ 融资/估值/背后公司
    ④ 技术路线差异（自研模型 vs 套壳 API）
```

**轻量跳过此步**，直接搜核心词。

### ② 执行搜索

- 每个子方向用不同搜索词，中英文都搜
- 优先看标题和摘要，快速筛出高相关度
- 轻量：WebSearch 1次 / 标准：WebSearch 2-3次 / 深度：每子方向1-2次

### ③ 质量过滤

每个信源过四问：
- **来源可识别吗？** 匿名帖子？自媒体？机构？学术？
- **有时效性吗？** 数据是什么时候的？还适用吗？
- **有利益冲突吗？** 厂商自述 vs 第三方评测 vs 竞品对比？
- **可交叉验证吗？** 别的独立信源也这么说吗？

### ④ 交叉验证

| 情况 | 处理 |
|------|------|
| 2+独立信源一致 | 标注为「确认」 |
| 仅1个信源 | 标注为「待验证」 |
| 信源之间矛盾 | 标注为「争议」，列出各方说法 |
| 无法找到可靠信源 | 标注为「信息不足」 |

**轻量跳过此步。** 标准做关键事实交叉。深度做全面交叉。

### ⑤ 综合输出

**统一输出格式（供下游工具消费）：**

```
核心回答
（直接回答用户问题的1-3句话）

关键发现
- 发现1（信源证据）
- 发现2

信源
- [来源名] | 可信度: 高/中/低 | 简要

不确定性
- 确认: ...
- 待验证: ...
- 争议: ...
- 信息不足: ...

升级建议（如有）
- 这个方向值得做深研？→ 建议启动深研
- 某个信源值得精读？→ 建议启动深读
```

## 四、元认知自检

| 检查项 | 自问 |
|--------|------|
| **来源偏向** | 我的信源有没有偏向某一方？ |
| **排序效应** | 我是不是只看搜索结果前几条就下结论了？ |
| **时效确认** | 引用的数据是现在的还是过时的？ |
| **确认偏误** | 我是不是在搜"支持我第一判断的证据"？ |
| **难度误判** | 我选的复杂度层级对吗？ |
| **升级遗漏** | 这个搜索结果有没有暗示一个值得系统研究的领域？ |
