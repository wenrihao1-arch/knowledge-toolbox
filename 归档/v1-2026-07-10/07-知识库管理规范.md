# 知识库管理规范

**定位：** 本文件是工具箱所有工具的共同约定。深读、深析、深研的归档规则统一按此执行。

---

## 一、归档路径

```
D:\知识库\                      ← 根目录
  ├── 工具箱\                   ← 本工具箱的定义文件，不动
  └── 资料库\                   ← 所有归档卡片和原文
        ├── 精读-标题.md         ← 深读产出
        ├── 深析-事件名称.md     ← 深析产出
        ├── 深析-事件名称-v2.md  ← 同一事件后续分析
        ├── 深研-领域名称.md     ← 深研产出
        └── （直接存放，不分子文件夹）
```

> 所有归档文件平铺在 `D:\知识库\资料库\` 下，不分子文件夹。文件名足够描述内容，配合标签检索。

---

## 二、文件命名规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 深读（精读） | `精读-标题.md` | `精读-AI存储芯片超级周期分析.md` |
| 深析（事件） | `深析-事件名称.md` | `深析-索尼PS5Pro涨价事件.md` |
| 深析-更新版 | `深析-事件名称-v2.md` | `深析-索尼PS5Pro涨价事件-v2.md` |
| 深研（领域） | `深研-领域名称.md` | `深研-存储芯片行业超级周期.md` |

**命名约定：**
- 标题不超过15个中文字符
- 不包含特殊符号（/ \ : * ? " < > |）
- 同一事件/领域的不同版本用 `-v2` `-v3` 区分

---

## 三、元数据行规范

每个归档文件的顶部必须包含一行元数据，格式如下：

```
类型: 深读 | 确定性: 高 | 信源来源: 搜索收集 | 关联: 深析-XX事件
```

| 字段 | 取值 | 说明 |
|------|------|------|
| **类型** | 深读 / 深析 / 深研 | 当前归档的类型 |
| **确定性** | 高 / 中 / 低 | 归档内容的可信度自我评估 |
| **信源来源** | 搜索收集 / 深读 / 深析 / 深研 / 用户直接提供 / 网络搜索 | 内容的原始信源来自哪里 |
| **关联** | 其他归档文件名（可选） | 和哪些旧归档有关联 |

## 四、统一标签体系

所有归档文件使用同一套标签。

### 主标签

| 标签 | 范围 |
|------|------|
| **#科技** | 技术、数码、互联网 |
| **#AI** | 人工智能、机器学习 |
| **#商业** | 经济、商业、创业 |
| **#社会** | 社会现象、文化 |
| **#健康** | 医疗、健康、养生 |
| **#效率** | 效率工具、学习方法 |
| **#科学** | 基础科学、研究 |
| **#读书** | 书评、读后感 |
| **#观点** | 评论、分析、深度文 |
| **#教程** | 经验分享、操作指南 |

### 额外标签（按需添加）

| 标签 | 用于 |
|------|------|
| **#事件** | 深析归档的额外标签 |
| **#观点文** | 精读观点类文章时使用 |

### 使用规则

- 每篇归档至少选 **1个主标签**
- 标签写在卡片底部：`━━━ 标签 ━━━ #科技 #商业`
- 不要自创标签，尽量复用现有体系

## 五、版本管理

### 深析版本规则

同一事件持续发酵时：

1. 不覆盖旧文件
2. 新建 `深析-事件名称-v2.md`（v3、v4……）
3. 新文件开头插入**时间线索引**：

```
> 📅 事件追踪时间线
> v1 (2026-07-01) → v2 (2026-07-05) → v3 (2026-07-09) [当前]
>
> 最新结论变化：之前认为XX → 现在认为YY
```

4. 同时回链到旧文件末尾追加 `后续分析：v2`

### 深研版本规则

深研文件不覆盖。有新结论时在文末追加更新章节：

```
## 🔄 v2更新（2026-XX-XX）

更新内容：
旧结论：XXX
新结论：XXX
```

### 过期文件标注

旧文件开头标注一句：

```
> ⚠️ 已过期 — 最新分析见 [新版文件名]
```

不要删除旧文件——时间线本身有价值。

---

## 六、关联回顾规则

每完成一次归档，快速检查是否和已有归档有关联：

- 同标签下有没有可串联的旧归档？
- 深读/深析/深研之间有没有交叉？
- 有强关联时在卡片中加一句 `🔗 关联: [文件名]`

---

## 七、升级来源记录

如果某次深析/深研是被其他工具建议升级而来的，在归档底部注明：

```
📌 启动来源: [搜索收集XX查询 / 深析XX事件]
```
