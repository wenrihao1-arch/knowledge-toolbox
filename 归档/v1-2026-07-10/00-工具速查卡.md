# 🗺️ 工具箱速查卡 — 一页决策

> 遇到问题不知道怎么下手？按这个流程走。

---

## 一、快速决策树

```
                   用户说了一句话
                        │
                        ▼
              有显式指令吗？
          "深读一下/深析一下/
           深研一下/搜一下"
                        │
          ┌───────┴───────┐
          ▼                ▼
        是                否
    直接路由到对应工具     看内容类型
                        │
              ┌─────────┼─────────┐
              ▼         ▼         ▼
           文章       事件       领域
         → 深读     → 深析     → 深研
              │
              ├── 具体查询 → 搜索收集 → 回答
              ├── 操作请求 → 直接执行
              └── 模糊/混合 → 追问2句
```

---

## 二、六个工具一句话定位

| 工具 | 一句话 | 输出 |
|------|--------|------|
| **需求解析** | 自动判断用户想干嘛，路由到正确工具 | 判定结果（静默） |
| **搜索收集** | 回答具体问题，搜完了给你汇总 | 信息汇总+信源 |
| **深读** | 精读好文章，提炼卡片归档 | 精读卡片 |
| **深析** | 拆解新闻/事件，看清背后逻辑 | 事件分析 |
| **深研** | 系统研究一个领域，整理成知识体系 | 研究成果卡片 |
| **工具箱附录** | 十三层分析技巧词典+12个工具展开 | 技巧参考 |

---

## 三、什么情况用什么工具

| 你想做…… | 用…… | 说明 |
|---------|------|------|
| 判断一个问题用哪个工具 | 需求解析 | 说人话就行，它会自动判定 |
| 查一个事实（"XX什么时候发布的？"） | 搜索收集-轻量 | 1次搜索搞定 |
| 对比几个东西（"A和B哪个好？"） | 搜索收集-标准 | 2-3次搜索+交叉验证 |
| 复杂查询（"这个赛道什么格局？"） | 搜索收集-深度 | 拆维度逐个搜 |
| 坐下来精读一篇长文章 | 深读 | 先评分，值不值得读 |
| 快速判断一篇短文要不要看 | 深读-扫读 | 不出卡片 |
| 分析一个正在发生的事件 | 深析 | 声明核查+立场矩阵 |
| 了解一个新领域 | 深研 | 四步法系统研究 |
| 写完分析后检查有没有漏洞 | 工具箱附录-元认知自检 | 找偏见和盲点 |

---

## 四、工具调用关系

```
需求解析（路由中枢）
  │
  ├──→ 搜索收集（信息收集）
  │
  ├──→ 深读（精读文章）←→ 深析（分析事件）←→ 深研（研究领域）
  │
  └──→ 直接回答 / 直接执行
```

> 深读/深析/深研内部可互相调用，但最多嵌套一层。

---

## 五、输出 - 什么情况该归档

| 工具 | 默认 | 你说"存一下" |
|------|------|-------------|
| 需求解析 | 不归档 | — |
| 搜索收集 | 不归档 | — |
| 深读 | 长文→卡片归档；短文不归档 | 写微卡 |
| 深析 | 不归档 | 写卡片到资料库 |
| 深研 | 归档 | — |
