# 深读体系 v4.5（Deep Reading System）

> **定位：** 三兄弟之一——深读管文章精读，深析管事件分析，深研管领域研究。
> **框架定位：** 十六层框架是参考工具箱，不是强制清单。每次精读从中选 3-5 个相关角度即可，不要机械跑满。

---

## 一、适用范围

输入按以下方式处理：
| 输入 | 处理方式 |
|------|---------|
| 完整文章链接 / 长文粘贴 | 选模板出卡片，归档 |
| 几句话 / 一段话 | 直接分析 — 对话中回复，不出卡片，不归档 |
| 一段话但值得存 | 你说"存一下"→ 写一个微卡归档 |

---

## 二、深读全流程（三阶段）

### 阶段①：读前筛选 — "值得深读吗？"

可选评分，也可凭直觉判断：

| 评分项 | 1分 | 3分 | 5分 |
|--------|-----|-----|-----|
| 信息密度 | 大部分已知/废话 | 有3-5个新信息点 | 每段都有新信息 |
| 信源可靠性 | 自媒体/无来源 | 有引用但未验证 | 引用权威来源/一手数据 |
| 思维增量 | 没有新启发 | 有1个新角度 | 颠覆了之前认知 |

**判定：** ≥10分值得深读，6-9分扫读，≤5分跳过。

**直觉优先原则：** 评分是辅助参考，不是硬性门槛。直觉觉得值得深读就直接进入。

**降级路径（≤5分时）：** 直接告知「这篇信息密度/信源质量偏低，不值得深读。需要我用搜索收集帮你快速查一下这个话题的相关背景吗？」

**读前设问：** 评分通过后、进入精读前，先写下 1-3 个问题：
> 我想从这篇文章中得到什么？
> 1. 
> 2. 
> 3. 

这步来自 SQ3R 和 KWHL 方法论的核心洞察：带问题读 vs 被动吸收，提取深度完全不同。

**扫读路径（6-9分）：**
```
扫读 | 标题

核心（1-2句）
要点（3-5个 bullet）
一句话思考
```

### 阶段②：精读分析（含辩证审视）

#### 模板选择

| 模板 | 适用 | 如何判断 |
|------|------|---------|
| **标准版** | 不确定类型、综合类文章 | 默认 |
| **新闻核查版** | 消息、快讯、纯报道 | 重点是"这事是真的吗" |
| **观点辩证版** | 评论、专栏、分析文 | 重点是"他在说服我什么" |
| **学术深读版** | 论文、研究报告 | 重点是"方法可靠吗" |
| **教程实操版** | 教程、指南、攻略 | 重点是"我能照着做吗" |

**当文章同时匹配多个模板时：** 新闻核查 > 学术深读 > 观点辩证 > 教程实操 > 标准版（按信息验证需求排序）。

#### 标准版卡片格式

```
━━━━━━━━━━━━━━━━━━━━━
 标题
━━━━━━━━━━━━━━━━━━━━━

读前设问回顾
（读之前提的问题，现在能回答了吗？哪些被回答了，哪些没有？）

核心观点
（1-3句概括全文核心）

事实核查 [真/伪/存疑] [依据]
（对关键声明做真伪判断，标注判定依据）

作者意图
（科普/商业推广/观点输出/新闻报道/技术分享/内容汇编）

要点提炼
├─ 原始亮点（直接击中我的段落/数据/观点，可附原文）
└─ 浓缩理解（用自己的话重新表述——不是原文概括，而是我真正学到了什么）

我的思考
（延伸理解和补充）

辩证看待
（选3-5个角度：适用边界、反面论证、作者立场、代价视角、时代局限、情绪检视等）

自我觉察
（我的情绪反应是什么？我希望这篇文章证明什么？我有没有因为立场而自动加减分？）

━━━━━━━━━━━━━━━━━━━━━
标签
━━━━━━━━━━━━━━━━━━━━━
```

**要点提炼的分级说明：**
- **原始亮点：** 直接摘录——读到某句话时被击中、愣住了、或者觉得"这个太对了"。
- **浓缩理解：** 用自己的话重新表述——不是 paraphrase 原文，而是"读完之后，我真正带走的东西是什么？"

**各模板的权重差异：**

| 模板 | 事实核查 | 要点提炼 | 辩证看待 | 自我觉察 |
|------|---------|---------|---------|---------|
| 标准版 | 均衡 | 均衡 | 均衡 | 均衡 |
| 新闻核查版 | 最重 | 轻 | 轻 | 轻 |
| 观点辩证版 | 中 | 中 | 最重 | 最重 |
| 学术深读版 | 最重 | 中 | 中 | 中 |
| 教程实操版 | 轻 | 最重 | 中 | 轻 |

**完成度自查（出卡片前过一遍）：**
- [ ] 读前设问回顾：读前提的问题现在能回答了吗？
- [ ] 核心观点：能用1-2句话说清这篇文章在讲什么
- [ ] 事实核查：关键声明做了真伪判断，不是照单全收
- [ ] 要点提炼：原始亮点和浓缩理解分开了吗？
- [ ] 辩证看待：至少选了3个角度审视
- [ ] 自我觉察：意识到自己读这篇文章时的情绪和立场
- [ ] So what?：读完这篇文章，我的认知/行动会有什么实际改变？

### 阶段③：归档留存

**回忆测试（归档前必做）：**
> 不看原文和卡片，凭记忆回答：
> 1. 这篇文章的核心观点是什么？
> 2. 我记住了哪 1-3 个具体信息？
> 3. 对照原文后，我遗漏了什么重要的东西？

**升级/降级记录：** 如果这篇文章是搜索收集找到的、或深析/深研过程中发现的，归档时在卡片下方注明 `发现来源: [搜索收集/深析XX事件/深研XX领域]`

**归档规则：**
1. 将精读卡片 + 原文全文写入一个 markdown 文件
2. 格式：精读在上，原文在下
3. 元数据行写在卡片顶部：`类型: 深读 | 确定性: 高/中/低 | 信源来源: ... | 关联: ...`
4. 归档到 `D:\知识库\资料库\`

## 四、工具间协作

```
深读（分析文章）
  → 读不懂/有疑问 → 调用深析
  → 涉及需要系统了解的领域 → 调用深研
  → 需要搜索补充信息 → 调用搜索收集

深析（分析事件）
  → 需要精读关键文章 → 调用深读
  → 感兴趣想系统了解 → 调用深研

深研（研究领域）
  → 遇到高质量文章 → 调用深读
  → 遇到相关新闻事件 → 调用深析
```

**注意：** "调用"不是"切换"——完成子任务后回到原任务。最终交付由你发起的指令决定。
**嵌套守卫：** 子工具调用最多嵌套一层。
