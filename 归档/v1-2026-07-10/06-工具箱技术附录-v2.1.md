# 工具箱技术附录 v2.1（Analysis Techniques Dictionary）

**定位：** 不是独立工具。需求解析、搜索收集、深读、深析、深研在运行过程中自动调用本附录中的技术定义。你不需要直接调用本文件。

---

## 一、十三层分析技巧

### 第1层 · 成本决策
被需求解析、深读阶段①、深析筛选、深研前置门控调用。
快速判断投入时间：扫标题、作者、开头结尾。不值得就跳过。放弃烂文章不是失败，是时间管理。

### 第2层 · 快速分类
被需求解析类型识别、深读模板选择、深析模板选择调用。
- 一手经验 vs 二手解读
- 观点文 vs 事实文
- 原创 vs 编译
- 领域属性

分类完成后阅读策略完全不同。

### 第3层 · 检查自身预设
被需求解析缺口识别、深读自我觉察、深析元认知、深研元认知调用。
读之前就意识到：我对这个话题有没有预设立场？我内心希望这篇文章证明什么？

### 第4层 · CERIC 论证拆解
被搜索收集、深读、深析、深研调用。

| 要素 | 问题 |
|------|------|
| Claim 主张 | 核心论点和结论是什么？ |
| Evidence 证据 | 哪些数据/事实/案例支撑它？ |
| Reasoning 推理 | 从证据到结论的逻辑是什么？ |
| Implications 影响 | 如果这是真的，意味着什么？ |
| Context 背景 | 这篇文章在什么背景下写的？ |

### 第5层 · 脱水拆骨架
同上。
去掉故事、案例、情绪渲染，只看逻辑骨架。用一两句话概括全文核心——做不到说明表达有问题或理解不够。

### 第6层 · 检查论据质量
同上。
- 来源追踪：能找到原始出处吗？
- 证据层级：系统综述 > RCT > 学术论文 > 行业报告 > 自媒体 > "我有一个朋友"
- 样本与代表性：样本量够吗？
- 时效性：数据还适用吗？

### 第7层 · 逻辑审查
被搜索收集、深读、深析、深研调用。
常见逻辑谬误：稻草人、滑坡、假因果、非黑即白、诉诸权威、诉诸情感。

### 第8层 · 作者/信源意图分析
被搜索收集、深读、深析、深研调用。
- 立场与利益
- 目标受众
- 隐性议程
- 输出一致性

### 第9层 · 逆向思考
被深读、深析、深研调用。
- 找反例
- 假设全反
- 不可证伪性检测

### 第10层 · 压力测试
被深读、深析、深研调用。
- 时间：5年前成立吗？5年后还会成立吗？
- 空间：在中国成立，在其他制度下呢？
- 领域：在A行业成立，在B行业呢？

### 第11层 · 多元思维模型交叉验证
被深读、深研调用。
- 心理学视角：有没有利用认知偏差？
- 经济学视角：激励机制是什么？
- 生物学视角：符合进化逻辑吗？
- 系统论视角：有没有二阶后果？

### 第12层 · 情绪雷达
被深读、深析、深研调用。
- 让我兴奋 → 是真有洞见还是说了我想听的？
- 让我反感 → 是真的不对还是在挑战我？
- 让我困惑 → 是表达不清还是我储备不够？

### 第13层 · 角色扮演
被深读、深析调用。
试着以作者/涉事方的第一人称回答质疑。区分"我真的理解了"和"我以为我理解了"。

## 二、工具箱展开

| 工具 | 用途 |
|------|------|
| **SIFT 四步** | 信息真伪快速验证：Stop→Investigate→Find better→Trace |
| **CRAAP 检验** | 来源质量五维度：时效/相关/权威/准确/目的 |
| **横向阅读** | 不在文章内部判断真伪，打开新标签页交叉验证 |
| **空白纸法** | 读前激活已有知识，读后凭记忆写总结 |
| **费曼技巧** | 用大白话把观点讲给外行人听。讲不清楚 = 没真懂 |
| **一周过滤器** | 读完一周后回忆：还记得什么？ |
| **能力圈原则** | 知道自己不懂什么，比知道自己懂什么更重要 |
| **ACH 竞争性假设** | 列出多个可能的解释 → 逐一检验 |
| **声明核查五级** | 属实/失实/误导/部分属实/证据不足 |
| **立场交叉矩阵** | 公开叙事×实际利益×权力筹码×行为逻辑×叙事可信度 |
| **Synthesis Matrix** | 子问题×多信源 → 交叉对比 |
| **MECE 分解** | 互不重叠、完全穷尽地分解问题 |
