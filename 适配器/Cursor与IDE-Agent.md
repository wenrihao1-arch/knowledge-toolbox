# Cursor 与 IDE Agent 适配器

这类 Agent 以代码编辑和当前工程上下文为中心。具体规则文件和功能会随产品版本变化，应通过当前版本支持的 Project Rules 或项目说明入口引用工具箱。

## 适用方式

- 代码、文档和仓库整理任务可以直接执行。
- 搜索只有在会话具备联网及网页读取能力时执行。
- 深读可以处理完整的本地材料；深析和深研按实际可访问证据执行，并明确是否经过外部核验。
- 若只能访问代码库，工具箱应退化为“分析本地材料”，而不是模拟外部研究。

## 推荐项目规则

```text
普通开发任务遵循仓库现有规范。
根据用户目标自动路由到对应核心协议，用户不需要说出工具名称。
不得根据模型内部知识声称已经完成实时搜索。
只有用户明确要求，或目标项目存在环境配置已记录的预授权时才归档。
不得加载工具箱/归档/中的旧版协议。
```

## 重点限制

- IDE 自动补全不等于 Agent 工作流，不能保证读取完整协议。
- 项目索引可能只提供文件片段；深读前确认是否拿到了完整材料。
- 修改知识库文件时应先读取环境配置，避免把内容写进当前代码项目的错误目录。
