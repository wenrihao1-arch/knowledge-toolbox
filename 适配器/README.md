# Agent 适配器

适配器负责把核心协议中的通用动作映射到具体 Agent。它不能修改核心质量标准，只说明平台如何读取规则、调用能力和处理限制。

## 通用动作

| 通用动作 | 含义 |
|---|---|
| `READ` | 读取用户材料、网页或工作区文件 |
| `SEARCH` | 搜索并打开外部来源 |
| `VERIFY` | 比较独立来源并核验关键事实 |
| `WRITE` | 在获准后写入目标知识库 |
| `ASK` | 仅在关键缺口会改变执行方向时追问 |
| `REPORT` | 说明结果、限制和实际完成状态 |

## 适配原则

1. 先确认当前会话实际提供的工具，不根据模型名称猜测能力。
2. 平台不支持某项动作时，按核心协议降级。
3. 不把平台专用工具名写回核心协议。
4. 不声称拥有跨对话记忆；只有平台明确持久化的文件或配置才算长期有效。
5. 平台系统指令、安全规则和用户当前指令优先。

## 接入层级

- 临时接入：在当前对话中上传或要求 AI 读取核心协议。
- 项目接入：由平台的项目说明文件指向本工具箱。
- 原生接入：封装为平台支持的 Skill、Rule、Command 或工作流。

项目接入和原生接入需要分别配置，不能假设一个平台的配置文件会被另一个平台读取。
