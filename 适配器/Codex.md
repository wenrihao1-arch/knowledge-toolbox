# Codex 适配器

## 接入方式

- 临时使用：先读取 `<工具箱路径>/README.md` 和当前任务对应的核心协议。
- 项目级使用：读取工作区根目录的 `AGENTS.md`；本仓库已经配置该入口。
- 可复用自动调用：将核心协议封装为 Codex Skill，并在 Skill 描述中写清触发条件。

仅把 Markdown 放进目录并不保证 Codex 在每个新任务中自动读取。

## 动作映射

| 通用动作 | Codex 中的执行 |
|---|---|
| `READ` | 使用当前会话提供的文件读取或网页访问能力 |
| `SEARCH` | 使用当前会话提供的联网搜索能力，优先打开原始来源 |
| `VERIFY` | 比较实际打开的独立来源，保留日期和适用范围 |
| `WRITE` | 在用户授权范围内使用文件编辑工具写入工作区 |
| `ASK` | 仅在关键选择无法从上下文判断时询问 |
| `REPORT` | 区分已完成、未执行和因能力不足而降级的部分 |

## Codex 注意事项

- 先遵守当前会话的系统、开发者和工作区规则。
- 不同 Codex 环境提供的工具可能不同，执行前以实际工具为准。
- 文件修改应保持现有仓库结构，不覆盖用户的无关改动。
- 需要最新 OpenAI 产品信息时优先使用官方 OpenAI 文档。
- 长任务应阶段性汇报，但不因例行步骤反复要求确认。

## 推荐入口规则

项目说明至少需要覆盖以下规则：

```text
根据用户目标自动路由到对应核心协议，用户不需要说出工具名称。
普通任务直接处理，不自动运行完整工具箱。
归档前读取 `<工具箱路径>/环境配置/当前环境.md`。
不得加载 `<工具箱路径>/归档/` 中的旧版协议。
```
