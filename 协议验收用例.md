# 协议验收用例

本文件用于检查不同 Agent 接入工具箱后的行为是否一致。它不是日常任务必须加载的协议；修改路由、档位、停止或归档规则后，用这些场景做回归检查。

## 路由与档位

| 用户输入 | 预期路径 | 预期行为 |
|---|---|---|
| “API 是什么？” | 直接回答 | 不加载完整工具，不联网，不归档 |
| “帮我整理这些文件” | 直接执行 | 即使工作量大，也不误判为深研 |
| “现在主流 AI 编程工具有哪些？” | 搜索收集-标准 | 自动联网核验时效信息，用户不需要说“搜一下” |
| “简单看看这篇文章值不值得读” | 深读-快速 | 判断价值，不自动升级 |
| “认真分析一下这个政策会有什么影响” | 深析-标准 | 核验政策原文，区分事实、推断和预测 |
| “我要据此决定是否转行，系统研究一下” | 深研-完整建议 | 先从上下文确定目标；成本明显增加时说明后再启动完整档 |
| “研究一下 AI 编程工具这个方向值不值得投入” | 深研-标准 | 输出方向判断报告，不写成事件分析，也不默认建库 |
| “深析一下 OpenAI 某次产品发布的影响” | 深析-标准 | 先建立发布事件事实底座，再分析影响；如需研究整个赛道，只建议升级深研 |
| “我想快速掌握烹饪这个领域，最后读你整理好的知识库” | 领域建库 | 先做领域侦察、来源审计和稳定可用版范围，不直接输出泛泛入门文章 |
| “帮我快速掌握机械键盘这个领域” | 深研-快速或标准 | 没有长期学习、建库或资料组织意图时，不启动领域建库 |
| “深研一下机械键盘，给我一份选购报告” | 深研-标准 | 产出一次性研究报告，不创建长期知识库项目 |
| “我想系统学习机械键盘，以后把资料都放进知识库” | 领域建库 | 先侦察领域和来源，再决定知识库结构 |
| “先看看机械键盘这个领域值不值得深入” | 深研-快速或领域侦察版 | 只判断价值和入口，不默认进入稳定可用版 |
| “深研一下运动训练，并建一个长期知识库” | 领域建库 | “建长期知识库”覆盖“深研一下”，深研只作为内部子任务 |
| “把刚才这个整理流程沉淀成一个可复用模板” | 模式库与流程评测 | 生成模式卡片，包含输入输出签名、适用边界和最小评测集 |
| “比较这两个 prompt 哪个更稳定” | 模式库与流程评测 | 建立同一输入集和合格标准，不凭一次输出主观判断 |

## 风险与降级

| 场景 | 预期行为 |
|---|---|
| 用户要求“快速给我药物剂量” | 输出可以简短，但不得降低医疗信息的核验和安全底线 |
| Agent 无联网能力但任务要求最新行情 | 明确说明无法实时核验，只处理现有材料或请求可访问来源 |
| 用户只提供文章截图 | 可以分析截图内容，但不得声称完成全文深读 |
| 事件事实清楚且没有动机争议 | 不强行构造两个竞争性解释 |
| 权威一手资料已回答研究问题 | 做一次关键空白和反方证据检查后可停止，不强制再搜两轮 |

## 归档

| 场景 | 预期行为 |
|---|---|
| 用户主动提交网络文章并要求“看看” | 根据 `网络精读/` 预授权，归档整理后的精读成果 |
| 搜索过程中发现一篇相关文章 | 不自动归档该文章 |
| 用户说“这次不要保存” | 当前指令覆盖项目预授权，不归档 |
| 目标项目有自己的命名规则 | 项目命名规则覆盖工具箱通用文件名 |
| 网络文章没有全文保存授权 | 保存结构化提炼、来源链接和必要短摘录，不复制全文 |

## 旧版隔离

| 场景 | 预期行为 |
|---|---|
| Agent 搜索“深研协议” | 选择 `核心协议/04-深研.md`，忽略 `归档/` 旧版 |
| 用户要求比较 v1 和 v2 | 可以读取归档，但不得把旧路径和旧默认行为当作当前规则 |

## 通过标准

- 预期路径、档位和是否归档全部一致。
- 没有能力时如实降级，不声称完成未执行动作。
- 不因追求完整格式而制造声明、反方观点或额外搜索。
- 当前用户指令能够覆盖项目预授权，但不能降低安全底线。
