# 分析方法

本文件是共用方法库，不是每次都要执行的清单。目标工具应根据问题选择最少且足够的方法，避免为了展示完整而增加无效篇幅。

## 方法选择

| 问题 | 优先方法 |
|---|---|
| 这段论证是否成立？ | 主张-证据-推理链、逻辑审查 |
| 这个来源可靠吗？ | 来源追踪、横向阅读、利益检查 |
| 为什么各方说法不同？ | 竞争性解释、利益与行为矩阵 |
| 一个领域由什么组成？ | 领域地图、依赖关系、证据矩阵 |
| 结论在什么条件下失效？ | 反例、边界测试、情景分析 |
| 我是否被自己的立场影响？ | 预设检查、情绪检查、反方检验 |
| 是否值得继续投入？ | 决策价值、信息增益、停止条件 |

## 主张-证据-推理链

把内容拆成：

- 主张：作者或分析者想让人相信什么？
- 证据：哪些数据、事实或案例支持它？
- 推理：证据为何能推出主张？
- 条件：结论依赖哪些前提？
- 影响：如果成立，对用户意味着什么？

证据真实不代表推理必然成立。

## 来源评估

检查：

- 能否追到原始出处？
- 来源是否具备相关专业能力？
- 数据和结论是否仍在适用时间内？
- 来源是否存在需要披露的利益关系？
- 其他独立来源能否验证关键事实？

“官方”说明来源身份，不等于天然正确；“第三方”也不等于天然独立。

## 横向阅读

不要只在来源自身页面内判断可信度。离开该页面，查询发布者背景、原始数据、独立评价和更好的替代来源。

## 竞争性解释

1. 为同一事实提出多个合理解释。
2. 列出每个解释应当预测到的现象。
3. 寻找最能区分这些解释的证据。
4. 优先淘汰与关键事实冲突的解释。

该方法用于减少“第一个听起来合理的故事”带来的锚定。

## 边界和压力测试

- 时间：结论在过去和未来是否仍成立？
- 地域：制度、文化和市场改变后是否成立？
- 人群：样本能否代表目标用户？
- 规模：从个例扩展到大规模时会发生什么？
- 激励：参与方的激励改变后，行为是否变化？
- 二阶影响：直接收益是否带来更大的后续代价？

## 证据矩阵

当存在多个子问题和来源时使用：

| 子问题 | 来源A | 来源B | 来源C | 综合判断 |
|---|---|---|---|---|

矩阵用于暴露空白和分歧，不是为了填满格子。

## 自我纠偏

在形成重要结论前检查：

- 我希望哪个结论成立？
- 是否只寻找支持初始判断的材料？
- 是否把重复转载误认为独立共识？
- 是否因为权威身份跳过了方法检查？
- 是否因为信息不足而补写了因果故事？
- 什么证据会让我改变判断？

自我纠偏应改变搜索或结论，而不只是输出一段仪式化的“自我觉察”。

## 记忆与学习

只有当任务目标包含学习时才使用：

- 费曼复述：不用原文术语解释核心概念。
- 空白回忆：关闭材料后写出核心观点和证据。
- 间隔回顾：在未来节点重新回答关键问题。
- 行动转换：把认知转成一个可观察的练习或项目。
