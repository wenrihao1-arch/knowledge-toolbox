# 搜索收集

搜索收集用于回答具体且可结束的问题。它追求足够可靠的答案，不追求把整个领域搜完，默认不归档。

## 何时必须搜索

满足任一条件时，应在 Agent 具备联网能力的前提下搜索：

- 信息可能近期变化，例如新闻、价格、政策、产品、软件版本和市场行情。
- 医疗、法律、金融等错误代价较高。
- 用户要求核实、引用、链接或指定来源。
- 推荐会让用户投入较多金钱或时间。
- AI 对事实没有把握，或无法确认内容是否过时。

稳定概念和无需实时信息的解释可以直接回答。

## 深度判断

搜索投入由以下因素共同决定，而不是机械规定搜索次数：

- 时效性：变化越快，越需要新资料。
- 风险：错误代价越高，验证越严格。
- 决策成本：用户投入越大，越需要比较与反证。
- 争议程度：各方分歧越大，越需要独立来源。
- 问题复杂度：子问题越多，越需要拆分搜索。

### 快速档

适合单一事实或初步判断。优先找到直接支持答案的一手来源，给出结论、日期和链接。

### 标准档

适合产品比较、工具选择、市场现状等问题。拆分关键维度，验证影响结论的事实，至少让重要结论得到一手来源或多个独立来源支持。

### 完整档

适合高风险或高投入决策。系统覆盖关键子问题、反方证据、时间边界和主要不确定性。若问题已经变成方向判断报告，应建议转入深研；若用户目标是长期学习、建库或持续更新，应建议转入领域建库。

## 信源原则

1. 优先原始文件、官方文档、论文、监管资料和当事方声明。
2. 第三方报道用于补充背景和独立核验，不能用大量转载制造“多方一致”。
3. 厂商资料可证明它公开声称了什么，不自动证明效果真实。
4. 新闻同时检查发布时间和事件实际发生时间。
5. 搜索结果摘要不是最终证据，应尽量打开原始页面。
6. 技术问题优先官方文档和原始研究；不同领域采用适合该领域的证据层级。

## 执行流程

1. 把用户问题改写成一句可验证的目标。
2. 拆出会改变结论的关键子问题。
3. 为每个子问题选择适合的来源类型和关键词。
4. 搜索并打开原始来源，记录日期与适用范围。
5. 比较来源是否独立，处理矛盾和利益关系。
6. 当新增搜索不再明显改变结论时停止。
7. 先回答问题，再给证据和不确定性。

## 证据标签

| 标签 | 含义 |
|---|---|
| 原始来源记录 | 原始文件或当事方来源直接记录了该声明或行为，但尚不代表效果与解释已经独立验证 |
| 独立确认 | 多个相互独立的可靠来源或直接证据支持同一事实 |
| 合理推断 | 由已知事实推导，但来源没有直接给出结论 |
| 有争议 | 可靠来源之间存在实质分歧 |
| 信息不足 | 当前资料不足以判断 |

不需要给每句话贴标签，但结论容易被误认为事实时必须区分。厂商公告通常只能标记为“原始来源记录”；除非存在独立证据，不能据此把产品效果标成“独立确认”。

## 输出结构

```text
核心回答
直接回答用户问题。

关键依据
- 对结论真正重要的证据及日期。

判断与边界
- 哪些是事实，哪些是推断；适用于什么时间和条件。

来源
- 只列实际使用并直接支持结论的来源。
```

快速档可压缩结构；完整档可增加方案比较、反方证据和决策建议。没有可靠答案时直接说明“目前无法确认”。
