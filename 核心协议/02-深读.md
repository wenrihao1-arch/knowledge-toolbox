# 深读

深读处理一篇文章、一份报告或一段完整材料，目标是理解作者说了什么、论证是否成立，以及用户真正值得带走什么。

## 边界

- 分析文本本身：深读。
- 分析文本所报道的现实事件：深析。
- 系统研究文本涉及的整个领域：深研。
- 将文本作为新领域知识库的资料入口，并持续建设专题和索引：领域建库。
- 材料中的事实需要外部核验：调用搜索收集，完成后返回深读。

## 开始条件

对整篇材料形成结论前，AI 必须能够读取完整材料。用户只要求分析段落、摘要或截图时可以直接处理，但必须明确结论仅适用于当前片段，不能冒充全文深读。

用户要求全文深读但当前只有摘要、截图残片或无法访问的链接时，应说明材料范围；缺失部分可能改变结论时，请用户补充内容。

不因标题或作者名假装读过正文。

## 投入档位

### 快速档

用于判断是否值得继续阅读：

- 一句话核心观点。
- 3-5个主要要点。
- 最明显的证据问题或适用边界。
- 是否值得进入标准档。

### 标准档

默认执行：

1. 说明文章试图回答的问题。
2. 提炼核心主张和论证链。
3. 区分作者提供的事实、解释与价值判断。
4. 核查会改变结论的关键声明，不做无意义的逐句核查。
5. 在适用时给出最强反例、适用边界和遗漏视角。
6. 总结用户值得记住或采取行动的内容。

### 完整档

在标准档基础上增加：

- 读前问题和阅读后回答。
- 来源追踪与关键事实的系统核验。
- 根据文章类型选择专用检查。
- 与已有知识或相关材料的比较。
- 回忆测试和归档准备。

## 类型检查

| 类型 | 重点 |
|---|---|
| 事实报道 | 来源、时间、遗漏背景和标题是否误导 |
| 观点文章 | 主张、证据、推理、反例和作者利益 |
| 学术材料 | 研究问题、方法、样本、统计解释和外推边界 |
| 教程指南 | 前提、步骤可复现性、成本、风险和失效条件 |
| 商业内容 | 目标受众、转化目的、选择性展示和真实价值 |

类型可以混合，但只选择真正相关的检查项。

## 标准输出

```text
核心观点
用1-3句话说明文章在主张什么。

论证结构
- 主要主张
- 关键证据
- 从证据到结论的推理

可靠性判断
- 可信部分
- 存疑部分
- 尚未核验部分

边界与反方
- 适用条件、反例、被忽略的代价或视角

真正值得带走的内容
- 认知更新
- 可执行行动（如有）
```

## 质量检查

- 是否真正读取了材料，而不是根据标题补全？
- 是否保留了作者原意，没有制造稻草人？
- 是否把“作者引用了来源”误当成“来源已经得到验证”？
- 是否为了显得辩证而强行编造等量反方观点？
- 最终输出是否比原文更容易理解，而不是更复杂？

## 整理提炼与保存原文

深读默认必须整理和提炼文章，包括核心观点、论证结构、关键证据、可靠性判断、适用边界、认知价值和必要短摘录。

“不默认复制全文”只限制原文复制，不限制整理提炼。网络文章默认保存分析和来源信息；只有材料属于用户、已获授权、处于公版，或用户明确要求且当前环境允许保存时，才考虑附全文。

允许保存全文时，原文优先进入目标项目的“原始资料”，深读成果进入“知识库”，两者通过链接关联。
