# 深研

深研用于围绕明确问题系统理解一个领域，或支持重要决策。它不是“尽可能搜得多”，也不是默认建设长期知识库，而是在明确目标下建立足够完整、可验证、可更新的认知结构，并产出一份能帮助用户判断方向的深度研究报告。

## 与深析的边界

深析处理具体事件，先问“发生了什么、谁参与、影响是什么”。深研处理领域或方向，先问“这个方向怎么看、是否值得投入、下一步怎么选”。

选择规则：

- 具体事件、政策、争议、产品发布：深析。
- 领域、赛道、职业方向、学习路线、工具栈、长期选择：深研。
- 事件能作为深研证据，但不能让深研变成只围绕单个事件的新闻分析。
- 领域研究中遇到关键事件事实不清时，可调用深析建立事实底座，完成后返回深研。

## 与领域建库的边界

深研回答的是：

- 这个领域/赛道/问题现在该怎么看？
- 我是否应该投入、购买、学习、转向或采取某个方案？
- 某几个路线、工具、观点或方案如何比较？
- 未来一段时间应该按什么路线继续研究或学习？

领域建库回答的是：

- 如何把一个陌生领域变成可长期阅读、可追溯、可更新的个人知识库？
- 资料如何筛选、审计、排序和入库？
- 书籍精读、专题笔记、术语表、索引和项目看板如何搭起来？

选择规则：

- 用户要“一次研究报告/路线/比较/决策建议”时，使用深研。
- 用户要“长期学习/建知识库/稳定可用版/以后读整理好的资料”时，使用领域建库。
- 领域建库可以把深研作为内部子工具，用来解决资料优先级、争议判断、路线选择等有界问题。
- 深研完成后可以建议归档或升级为建库，但未经用户同意不自动创建长期项目结构。

## 启动定义

正式研究前确定四件事：

1. 决策或学习目标：研究结果要帮助用户做什么？
2. 范围：包含什么，不包含什么？
3. 交付物：地图、比较、学习路线、方案还是决策建议？
4. 停止条件：达到什么证据和覆盖度即可结束？

若上下文已经给出答案，AI 可直接开始；只有缺口会实质改变研究方向时才追问。

## 研究验收标准

启动时同时定义：

- 必须回答的关键问题。
- 关键结论需要达到的证据强度。
- 用户需要的最终交付物。
- 信息截止日期或适用时间范围。
- 可以接受保留的未知项。

验收标准用于决定何时停止，不能用固定搜索次数或来源数量代替。

## 投入档位

### 快速档

建立初步地图、主要概念、关键参与者、争议和进一步研究建议。适合判断是否值得投入，或判断是否需要升级为领域建库。

### 标准档

围绕明确目标建立领域地图，覆盖关键分支和可信来源，比较主要观点，形成可执行结论。默认用于学习规划和中等成本决策。

### 完整档

用于高投入决策或复杂研究。在标准档基础上增加可复现的搜索记录、系统证据矩阵、历史演化、反方检验、情景分析和更新信号。若目标已经变成长期知识库建设，应转入领域建库，而不是把深研无限扩张。

## 研究流程

### 1. 定义问题

将宽泛主题改写为一个主问题和若干子问题。分支应尽量完整，但不强求形式上的 MECE；真实领域允许交叉和依赖关系。

### 2. 建立初始地图

- 核心概念与第一性约束。
- 主要分支和依赖关系。
- 与用户目标最相关的重点。
- 当前未知和可能争议。

### 3. 设计证据策略

为不同子问题选择合适来源。证据层级必须适应领域：医学、软件、行业、政策和历史研究不能共用一条固定排序。

### 4. 迭代搜索与综合

每轮执行：

```text
提出子问题 -> 找原始来源 -> 提取证据 -> 比较分歧
-> 更新地图和判断 -> 判断是否继续
```

新增材料只有在能填补关键空白、验证争议或改变结论时才纳入。来源数量不是质量目标。

### 5. 处理分歧

1. 检查来源是否讨论同一问题和同一时间范围。
2. 比较方法、数据、利益关系和适用边界。
3. 寻找更高质量的一手资料或独立仲裁。
4. 无法解决时保留“暂无共识”，不强行统一。

### 6. 综合与行动

最终结论必须回答研究目标，并说明：

- 当前最可靠的认知是什么？
- 哪些结论只在特定条件下成立？
- 用户下一步应该做什么？
- 什么变化会让结论过时？

## 停止条件

判定为“完成”或“暂时完成”前，检查以下条件：

1. 研究目标已经得到直接回答。
2. 所有必须回答的关键子问题都有结论；无法回答的项目已明确标注，且不会阻止当前用途。
3. 会影响决策的关键结论具有足够可靠的证据。
4. 重要争议已经解释；无法解决的分歧被保留为“暂无共识”。
5. 已针对关键空白、反方证据和主要争议做过一次饱和检查，新增材料不再实质改变核心判断。
6. 已经能够生成约定的交付物和下一步行动。
7. 剩余未知不会阻止用户当前决策，并已说明未来更新信号。

快速档或权威一手资料已经直接回答问题时，一次有针对性的饱和检查即可。完整档可以进行多轮迭代，但不以固定轮数作为完成条件。无法联网时，用当前可访问材料完成同类空白检查，并标注证据范围。

停止状态必须区分：

| 状态 | 含义 |
|---|---|
| 完成 | 达到研究目标和验收标准 |
| 暂时完成 | 当前信息足够行动，但需要在未来节点更新 |
| 信息不足 | 关键证据无法获得，不能形成可靠结论 |
| 范围受限 | 因权限、时间、资源或用户限制停止 |

停止不是宣称领域已经被彻底研究，而是说明当前研究已经达到约定用途。

## 证据矩阵

标准档和完整档按需使用：

| 子问题 | 来源与日期 | 证据 | 分歧 | 当前判断 | 置信度 |
|---|---|---|---|---|---|

每个关键判断尽量由直接证据或多个独立来源支撑。没有覆盖的子问题进入“后续待研究”，不能靠常识补齐。

## 标准交付

```text
研究目标与范围

方向判断结论

推荐方向 / 不推荐方向 / 暂缓判断

领域地图

关键发现与证据

主要争议和不确定性

对用户的行动建议与执行路线

后续待研究与更新信号
```

研究过程较长时，应阶段性汇报已完成内容、当前判断和剩余问题；只有范围需要扩大或成本明显增加时才等待用户决策。

## 升级与交接

深研输出中如果发现值得长期学习或建库，应只给出升级建议和最小交接包，不直接改造成建库项目。

最小交接包：

```text
建议升级原因：
建议建库目标：
已确认的核心问题：
候选资料类型：
关键风险或争议：
建议第一阶段：领域侦察版 / 稳定可用版
```
