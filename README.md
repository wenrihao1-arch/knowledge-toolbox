# 个人知识处理工具箱

这是一套与具体模型无关的知识处理协议。它规定 AI 应该达到什么结果，但不假设 AI 一定拥有联网、文件读写、子 Agent 或长期记忆能力。

## 设计原则

1. 用户用自然语言表达目标，AI 自动选择工具和投入档位。
2. 用户指令优先，并可随时覆盖 AI 的选择。
3. 普通对话不自动套用完整流程，默认使用能解决问题的最低投入档位。
4. 事实、推断和建议分开表达。
5. 没有执行能力时明确降级，不假装搜索、读取或写入成功。
6. 只有用户明确要求保存，或目标项目已经记录明确预授权时才归档。
7. 核心协议不写死平台工具名称和本机路径。
8. 面向公开共享设计：核心规则应让不同用户、不同 Agent、不同工作区都能理解和执行。

## 三层结构

| 层级 | 目录 | 作用 |
|---|---|---|
| 核心协议 | `核心协议/` | 定义调用条件、质量标准、输出结构和停止条件 |
| 模式库 | `模式库/` | 保存可复用模式卡片和模板 |
| Agent 适配器 | `适配器/` | 将通用动作映射到不同 Agent 的实际能力 |
| 环境配置 | `环境配置/` | 记录当前工作区、知识库目录和归档路由 |
| 参考资料 | `参考资料/` | 记录外部工具借鉴，不覆盖核心协议 |

协议修改后的回归检查见 `协议验收用例.md`。

## 核心工具

| 文件 | 处理对象 | 默认行为 |
|---|---|---|
| `00-调用协议.md` | 用户意图和任务分流 | 静默选择最轻可行路径 |
| `01-搜索收集.md` | 具体且可回答的问题 | 搜索、验证、回答，不归档 |
| `02-深读.md` | 一篇文章或一份材料 | 理解并审查文本 |
| `03-深析.md` | 现实事件、政策或争议 | 分开事实、推断和预测 |
| `04-深研.md` | 有明确问题边界的领域研究或决策研究 | 产出一次性研究报告、地图、比较或路线建议 |
| `05-分析方法.md` | 共用分析技术 | 按需调用，不机械跑满 |
| `06-知识库归档.md` | 保存和版本管理 | 读取环境配置后归档 |
| `07-领域建库.md` | 新领域学习和长期知识库项目建设 | 编排侦察、资料审计、知识库结构和稳定可用版 |
| `08-模式库与流程评测.md` | 可复用 AI 模式和流程质量 | 把成功用法沉淀成模式卡片，并用验收集评测 |

## 职能边界

| 用户真实目标 | 应用工具 | 交付物边界 |
|---|---|---|
| 回答一个具体问题 | 搜索收集或直接回答 | 答案、依据、边界 |
| 理解一篇文章、视频稿、论文或报告 | 深读 | 对该材料的结构化理解和可靠性判断 |
| 分析一个现实事件、政策、争议或产品发布 | 深析 | 事实链、解释、预测和不确定性 |
| 对某领域做一次系统研究，服务一个决策或学习计划 | 深研 | 一次性研究报告、领域地图、比较、路线或建议 |
| 把一个陌生领域建设成以后能持续阅读、调用和更新的知识库 | 领域建库 | 项目结构、来源审计、专题笔记、稳定可用版、后续加强机制 |
| 把反复成功的 AI 用法沉淀成可复用模板，或评测工具箱是否稳定 | 模式库与流程评测 | 模式卡片、输入输出签名、最小评测集、回归结果 |

`深研` 和 `领域建库` 的区别：

- 深研是研究工具，默认不创建长期项目结构。
- 领域建库是项目编排工具，可以在内部调用搜索收集、深读、深研和归档。
- 如果用户既说“研究一下”，又说“建知识库、以后读你整理好的、长期学习、稳定可用版”，优先走领域建库。
- 如果用户只是要判断一个方向是否值得做，先用深研快速档或领域建库的领域侦察版；不要直接启动完整建库。

`深析` 和 `深研` 的区别：

- 深析处理一个现实事件、政策、争议、行业动作或产品发布，核心交付是事实底座、参与方、影响和后续信号。
- 深研处理一个领域、赛道、方向或复杂选择，核心交付是方向判断报告、比较、路线和决策建议。
- 一个事件可以触发深研建议，但深析本身不扩写成领域教材。

## 调用方式

- 普通提问：直接回答或执行，不启动完整工具。
- `搜一下……`：调用搜索收集。
- `深读一下……`：调用深读标准档。
- `深析一下……`：调用深析标准档。
- `深研一下……`：先从上下文确定研究目标，产出一次性研究报告；关键目标缺失时再询问。
- `我想长期学习/系统掌握一个新领域，并建立知识库……`：调用领域建库，先做领域侦察和资料审计；只有用户要长期学习或建库时才进入稳定可用版建设。
- `帮我快速了解/掌握一下某领域……`：若没有建库、长期学习或大量资料组织意图，优先用深研快速档或标准档。
- `快速……`、`标准……`、`完整……`：显式指定投入档位。
- `存一下`、`归档`：完成当前任务后调用归档协议。
- `把这个流程沉淀成模板/模式`、`给这个工具做验收用例`、`比较两个 prompt 哪个稳定`：调用模式库与流程评测。

AI 负责默认路由，用户不需要记住工具名称。AI 可以建议升级，但不能未经用户同意把轻量任务升级成长时间研究；自动归档只在环境配置记录的预授权范围内执行。

## 能力降级

| 当前能力 | 执行方式 |
|---|---|
| 可联网、可读写文件 | 执行所选档位，并在获准后归档 |
| 可联网、不可写文件 | 完成搜索与分析，输出可归档内容 |
| 不可联网、可读用户材料 | 仅分析现有材料，标注未联网核验 |
| 无法读取材料 | 请求用户提供内容或可访问位置 |

具体平台差异见 `适配器/`，当前仓库路径见 `环境配置/当前环境.md`。

## 共享发布原则

- 核心协议只写通用规则，不写个人路径、个人偏好或特定软件绑定。
- 个人工作区、默认归档目录和模型分工写入 `环境配置/` 或项目级说明。
- 示例应覆盖正例和反例，特别是深析/深研、深研/领域建库的边界。
- 适配器只说明不同 Agent 如何执行同一协议，不改变协议本身的质量标准。
- 可复用流程应沉淀成模式卡片，并至少配一组最小评测用例。
